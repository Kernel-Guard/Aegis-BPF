# Maintainers

This file lists the current maintainers of AegisBPF, their scope, and how
to reach them. Maintainers have merge rights, release authority, and
security triage responsibility as defined in [`GOVERNANCE.md`](GOVERNANCE.md).

## Seeking co-maintainers

AegisBPF is actively seeking additional maintainers from **different
organizations** to meet CNCF Incubation requirements and to distribute
release, review, and security-triage load. Strong candidates demonstrate:

1. Sustained high-quality contributions across BPF programs, daemon,
   operator, or CI/testing.
2. Ownership of a subsystem (for example: operator & webhooks, network
   policy, policy compiler, fuzzing, performance gates).
3. Active code review with a pattern of timely, thoughtful feedback.
4. Availability to respond to release and security matters on a
   predictable cadence.

If you are interested, please open a GitHub Discussion titled
"Maintainer candidacy: *your-name*" summarizing your contributions and
desired scope. Existing maintainers will respond within two weeks.
